Introduction

The purpose is to detail the implementation of the requirement as defined in the Software Requirements Specification. Design components are described in detail. High level component design is also added which shows which component mainly depends on what other components.Moreover,in this software design specification class diagram is added with class description which are described in detail such as name,type ,purpose of every attribute of each class.Furthermore, database design are also added.

Document Conventions

  • In this software design specification, the primary key is written in bold font.

Description of Design Components

Class Diagrams

Detailed Class Diagram(s)

Detailed Class Description(s)

Name of Class: User

Purpose: For storing data of every individual maintaining uniqueness for each of them. This information is stored after a user is registered into the system. And further these data are used for login purpose

Name Purpose Type
token The token is stored for every user for maintaining uniqueness for every individual . String
nsuId For storing the NSU ID of every user as the user must be from North South University. String
fullname For storing the full name of a user for registration purposes. String
email Email of a user is stored for registration and login purposes and also for sending necessary emails to the user from the system. String
authType For identifying whether a user Is registered through google sign up or by providing a password manually. AuthType
password For storing the password of the user which he/she will use to login to the system. String
userType To identify whether the registered user is a student,teaching assistant,faculty ,helper or an admin. UserType
ActorType A user can be a reviewer,non-reviewer or a sys-admin. ActorType
scannedNsuId Nsu Id of the user should be scanned and submitted at the time of registration for proper identification of the user. ActorType
status Status can be verified ,not verified or diabled. The default value is not verified. String

Name of Class: UserResetPassword

Purpose: If a user forgets his/her password which was provided at the time of registration,he/she needs to reset the password to login into the system. For resetting passwords, users will be provided with a link through email which will expire after 24 hours.

Name Purpose Type
token For maintaining uniqueness of every individual ,a token is stored String
used To identify whether the password is reset or not through the link which was provided to the user via email. String
expiresAt The link which will be provided to the user through email will expire after 24 hours.The user needs to reset the password before that time. Date
status The default value of status is Active. String

Name of Class: UserVerification

Purpose: At the time of registration, after providing all the necessary information,user will be provided with a link via email for verification whether all the information of the user is valid or not.Verification is needed to complete the registration.

Name Purpose Type
token For maintaining uniqueness of every individual, a token is stored String
isVerified To identify whether the user is verified or not.User must be verified for complete the registration process. String
expiresAt The link for user verification which will be provided to the user through email will expire after 24 hours.The user needs to be verified before that time. Date
status The default value of status is Active. String

Name of Class: UserSession

Purpose: The Purpose is to keep a user logged in to the system for 7 days if she/he does not log out to the system. After 7 days, the user will be logged out of the system.

Name Purpose Type
token For maintaining uniqueness of every individual, a token is stored String
expiresAt A user will be logged out after 7 days. He/she will be logged in for 7 days into the system. Date
status The default value of status is Active. String

Name of Class: Complaint

Purpose: The purpose of the class is to store the complaint lodged by the user.The complaint can be edited by the user anytime .Also the edit history will be stored.The complaint can also be reviewed by the assigned reviewer only.

Name Purpose Type
token For maintaining the uniqueness of every complaint, a token is stored. String
description The description of the complaint lodged user will be stored.The description of the complaint must be less than or equal to 1024 characters. Date
status The default value of status is Active. String
version The complaint can be edited by the user and the edit history can be shown with date and time of editing. String

Name of Class: CompaintAgainst

Purpose: It's a relational table that stores the information of a user as a complainee.

Name Purpose Type
token For maintaining the uniqueness of every relation, a token is stored. String
description The complainet lodger can be edited by the user.User can add more person against whom complaint has been lodged.Again lodger can also remove a complaineeThe edit history can be shown with date and time of editing. Integer
status To store the status of the class.The default value of status is Active. String

Name of Class: CompaintComment

Purpose: The comment which is made by the reviewer is stored.A reviewer can comment multiple times.Again if a reviewer changes the reviewer ,the update will also be stored as a comment.

Name Purpose Type
token For maintaining the uniqueness of every user, a token is stored. String
comment A reviewer can add comments in a complaint multiple times.Again if a reviewer reassigns the reviewer ,the update will also be shown as a comment.All these comments are shown accordingly. String
status To store the status of the class.The default value of status is Active. String
commentType To store the information whether the comment is an actual comment made by the reviewer or it is an update that the reviewer of the complaint has been changed. String

Name of Class: CompaintEvidence

Purpose: The purpose is to store the evidence of the complaint which was made.Without proper documentation, a lodger cannot lodge a complaint against another user.

Name Purpose Type
token For maintaining the uniqueness of every user, a token is stored. String
fileName It stores evidence by giving it a unique file name. String
originalFileName It is the original file name which the lodger has uploaded. String
status To store the status of the class.The default value of status is Active. String
version The evidence can be edited by the lodger any time. He/she can add or remove any evidence again & edit history will be shown with versioning. Integer

Database Design

ER Diagram

ER Description (Name, Type, Purpose of Foreign Key)

Name of Class: UserResetPassword

  • Attribute name: userToken

    Type: String

    Purpose: It associates with the user i.e only a user can reset the password.

Name of Class: UserSession

  • Attribute name: userToken

    Type: String

    Purpose: It handles whether the user is logged in to the system or not and this user is associated with the userToken.

Name of Class: UserVerification

  • Attribute name: userToken

    Type: String

    Purpose: It handles whether the user is verified via email and the user is associated with the userToken.

Name of Class: Complaint

  • Attribute name: userToken

    Type: String

    Purpose: The user who will lodge the complaint is associated with the userToken.

  • Attribute name: reviewerToken

    Type: String

    Purpose: A reviewer is assigned for each complaint.The user who will review the complaint is associated with the reviewerToken.

Name of Class: ComplaintAgainst

  • Attribute name: complaintToken

    Type: String

    Purpose: It belongs to the complaint part.Mainly,it associates the user with the complaint.

  • Attribute name: complaintAgainstToken

    Type: String

    Purpose: It associates with the user against whom a complaint is being lodged.

Name of Class: ComplaintComment

  • Attribute name: userToken

    Type: String

    Purpose: It associates the user i.e the reviewer of the complaint who makes the comment.

  • Attribute name: complaintToken

    Type: String

    Purpose: It identifies which comment belongs to what complaint.

Name of Class: ComplaintEvidence

  • Attribute name: complaintToken

    Type: String

    Purpose: It recognizes which evidence belongs to what complaint.
